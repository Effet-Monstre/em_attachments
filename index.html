EmAttachmentsFile attachments for Elixir

Upload to cache, promote to store, run plugins, save metadata — inspired by Shrine for Rails.

Get Started
View on GitHub

Two-phase lifecycle

Files land in cache on upload and move to permanent storage on save — always via a clean, auditable pipeline.

Plugin system

MIME detection, dimension validation, and derivative generation hook into every phase without coupling your uploader code.

Ecto-native

Each uploader is an Ecto.Type — use cast_attachments/3 in your changeset like any other field.

Multiple backends

Local filesystem for development, S3 (no ExAws) for production, or bring your own backend with a five-function behaviour.
